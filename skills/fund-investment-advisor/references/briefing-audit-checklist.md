# 简报系统审计清单

2026-07-08 使用 ai-berkshire 四大师框架对基金投顾系统做全面审计后发现的问题与修复方案。

## P0 级（数据错误，必须立即修复）

### 1. 持仓数据不同步
- **现象**：简报显示已清仓的基金（022184 仓位34.9%、026211 仓位2.9%），但截图显示已赎回
- **根因**：advisor.py 可能读取了缓存数据而非 holdings 表最新数据，或 DB 更新后未触发 advisor.py 重新加载
- **修复**：检查 advisor.py 的数据读取路径，确保每次运行都从 `holdings` 表读取最新数据

### 2. 定投信息过期
- **现象**：简报显示"富国全球科技(QDII)C | 每日定投 | 200元/日"，但实际已切换到017731嘉实全球产业升级每日100元
- **根因**：cron 任务 `29e8f6e83210`（022184 DCA更新）仍在运行，未切换到017731
- **修复**：
  1. 停掉旧 cron：`hermes cron pause 29e8f6e83210`
  2. 创建 017731 的 DCA cron（每日20:00，100元）
  3. 更新 advisor.py 中的 DCA 显示信息

### 3. 周报基金名称缺失
- **现象**：`weekly_review_latest.json` 中 `fund_name` 全部是 "基金005165" 格式
- **根因**：周报生成脚本未从 holdings 表读取 fund_name，只用了 fund_code 拼接
- **修复**：在周报生成逻辑中，`SELECT fund_code, fund_name FROM holdings` 替代 `SELECT fund_code`

### 4. 月度评估回测全零
- **现象**：`monthly_evaluation_latest.json` 中 `total_trades=0`，`final_value=initial_capital`，回测完全没执行
- **根因**：cron 运行时报 `Broken pipe` 错误，可能是 Python 输出被管道截断
- **修复**：检查 `cron_monthly_evaluation.sh`，移除可能导致管道截断的命令（如 `head`、`tee`）；确保 Python 进程不被 SIGPIPE 终止

## P1 级（逻辑缺陷）

### 5. 风控评分新建仓豁免
- **现象**：017731 嘉实QDII 刚开始定投（收益0%），风控评分48分建议"考虑减仓"
- **修复**：在风控评分中加入 `first_buy_date` 判断：
  ```python
  days_held = (datetime.now() - first_buy_date).days
  if days_held < 90:
      risk_suggestion = "持有观察（新建仓不足3月）"
  ```

### 6. 移动止盈措辞混淆
- **现象**："触发移动止盈（阈值8.0%）"容易误读为"立即全部卖出"
- **修复**：改为"触发分批止盈第一步（减仓1/3）"或"触发移动止盈信号，建议分批操作"

### 7. 组合级行业穿透
- **现象**：行业配置只按单只基金展示，无组合级行业暴露度
- **修复**：利用 `holdings_overlap.py` 的穿透数据，在持仓概览后加一段：
  ```
  组合行业暴露度:
    通信: 40%+ (4只基金持有)
    半导体: 25%+ (3只基金持有)
    光模块产业链: 15%+ (5只基金持有)
  ```

### 8. 持仓重叠度→操作建议
- **现象**：已识别天孚通信4只基金持有（合计27%），但无操作建议
- **修复**：对严重重叠（3+基金）的个股，自动生成减仓建议

## P2 级（质量提升）

### 9. 无基准对比
- 加沪深300/创业板指日度收益对比，让用户知道跑赢/跑输

### 10. 无估值分位
- 加持仓核心个股的 PE/PB 分位。PB 分位 >95% 时自动预警

### 11. 均线策略信号标注
- 模拟信号必须明确标注"⚠️ 策略模拟信号，基于技术面回测，非实际持仓操作"
