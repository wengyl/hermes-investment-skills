# Holdings Overlap Analysis Module

Module: `holdings_overlap.py` — 持仓重叠度分析

## 功能
- 获取所有持仓基金的前十大重仓股（API + 本地缓存）
- 识别跨基金重叠持股（2只=中度，3只+=严重）
- 计算板块集中度（优先用基金自带行业配置数据）
- 生成持仓优化建议（独立赛道 vs 高重叠）

## 关键设计决策

### 1. 缓存策略
- 东方财富 API 有限频，不能每次报告都调
- 缓存表：`fund_holdings_cache`（fund_code PRIMARY KEY, top_holdings JSON, industry_allocation JSON, updated_at）
- 过期时间：30天（重仓股数据按季度更新）
- API 失败时返回推断数据但**不缓存推断结果**

### 2. 板块集中度计算
- **优先用 `industry_allocation`**（基金自带的行业配置），不用从股票名推断
- `_classify_stock_industry()` 准确率低（很多股票被归入"其他"）
- 过滤掉"其他"类别，只展示有意义的板块

### 3. 种子数据
- `seed_known_holdings()` 方法：当 API 不可用时，从预置数据导入
- 数据来源：东方财富基金持仓快照
- 只在缓存为空时写入，不覆盖已有数据

### 4. 集成方式
- `advisor.py` 早报和晚报都集成
- 轻量运行：无重叠时不显示，失败不影响主报告
- 重叠分析作为独立 section 插入在持仓概览之后

## 依赖
- `data_fetcher.py`：`get_fund_holdings()` API 调用
- `db_init.py`：`get_db_path()` 数据库路径
- SQLite 表：`fund_holdings_cache`

## 踩坑记录
1. API 返回空 top_holdings → 需要缓存 + 种子数据兜底
2. `_classify_stock_industry` 把大量股票归入"其他" → 改用 fund-level industry_allocation
3. 板块集中度百分比是**跨基金累加**（同一只股票在不同基金的行业配置会叠加），所以可能>100%

## _classify_stock_industry 扩展 (2026-06-05)

**问题**: 旧版只覆盖10个行业~40个关键词，大量科技股(北京君正、源杰科技、拓荆科技等)被归入"其他"，导致行业配置无意义。

**修复**: 扩展到17个行业~120个关键词。新增行业：
- 半导体: +北京君正, 源杰, 拓荆, 长光华芯, ASMPT, SK海力士, 英特尔, 阿斯麦, 海力士, 美光, 兆芯, 龙芯, 景嘉微, 紫光
- 通信: +新易盛, 中际旭创, 天孚通信, 长芯博创, 长飞光纤, 永鼎, 德科立, 盛科通信, 光迅, 烽火, 亨通, 中天, 东山精密
- 电子: +胜宏科技, 炬光科技, 麦格米特, 环旭, 工业富联, 立讯, 莱特光电, 斯迪克, 伟创电气, 震裕科技, 恒辉安防 等
- 互联网: +谷歌, Facebook, Meta, 亚马逊, 微软, 苹果, Netflix
- 电力设备: +伟创, 恒立液压, 斯菱
- 机器人: +五洲新春

**注意**: 分类器是"关键词匹配"而非行业数据库，仍有局限。对于海外股票(台积电、英伟达等)效果好，但冷门A股可能仍归入"其他"。
